trailsegments.geojson

Name Type Req'd? Summary Values
Geometry Linestring or MultiLineString yes The geometry object which describes the trail segment, per the data type as defined in the geoJSON specification. n/a
id string yes A unique identifier assigned by the data producer. The “Named Trails” described in namedTrails.geojson are constructed with arrays of trail segments, identified via this field. n/a
stewardId string yes id of agency or organization responsible for trail maintenance, regulation, and safety. n/a
name string Some segments have proper names of their own. This allows data providers to name segments, which are in turn accepted in OpenStreetMap n/a
highway string yes An OSM-compliant attribute that determine whether a path is treated as a “trail”. "path"
vehicles string A judgement as to the appropriate use of OHV, ATVs, and snowmobiles on this trail segment. "yes", "no", "permissive", "designated"
foot string Indicates whether walking and hiking is an appropriate use of this trail segment. This field is derived from OpenStreetMap conventions. "yes", "no", "permissive", "designated"
bicycle string Indicates whether general bicycling is an appropriate use of this trail segment. This field is derived from OpenStreetMap conventions. "yes", "no", "permissive", "designated"
horse string Indicates whether horseback riding or equestrian travel is an appropriate use of this trail segment. This field is derived from OpenStreetMap conventions. "yes", "no", "permissive", "designated"
ski string Indicates whether over-land or cross-country skiing is an appropriate use of this trail segment. This field is derived from OpenStreetMap conventions. "yes", "no", "permissive", "designated"
wheelchair string Indicates whether this trail segment is considered wheelchair accessible. This field is derived from OpenStreetMap conventions. Other attributes that describe accessibility can be included in the 'osmTags" field. "yes", "no", "permissive", "designated"
osmTags string-encoded array This field is designed to incorporate a fungible number of key-value pairs that describe segments within OpenStreetMap. Anticipated use of this field includes the incorporation of keys such as "width", "surface" and "mtnbike". n/a

namedtrails.csv

Name Type Req'd? Summary Values
name string yes The proper name of the trail, as advertised to visitors and public communications.
segmentIds string-encoded array yes An array of IDs, which allows applications to construct "Named Trails" from a collection of one or more trail segment geometries.
id string yes A unique identifier assigned by the data producer.
descriptn string yes This is the primary field for narrative descriptions of a Named Trail, and the experience or features a visitor might expect to encounter.
dsgnations string-encoded array This field allows for data contributors to indicate whether a given trail has been bestowed with particular nationl, state or local designations or distinctions.
partOf string-encoded array This field allows for data contributors to indicate a relationship between Named Trails that share constiuent segments. The anticipated use case is to indicate that a trail serves a leg in a larger regional or national trail.

trailheads.geojson

Name Type Req'd? Summary Values
geometry string yes The point geometry object which describes the trail segment, per the data type as defined in the geoJSON specification.
name string yes The public name of the trailhead, as advertised to visitors.
trailIds string yes A collection of IDs for Named Trails (match to "id" in namedtrails.json), each representing a trail that can be accessed via this trailhead.
stewardIds string-encoded array yes A collection of IDs for Stewards (match to "id" stewards.json), representing each agency or organization responsive to visitor concerns or inquiries about this trailhead.
parkName string A collection of park names that visitors should associate with each trailhead. These park names currently exist independently from other aspects of the specification.
address string The street address, if one exists, that is associated with this trialhead.
parking string An indication as to whether visitors can expect to find automobile park at or immediately adjacent to this trailhead.
drinkwater string An indication as to whether visitors can expect to find drinking water available at or immediately adjacent to this trailhead.
restrooms string An indication as to whether visitors can expect to find restrooms at or immediately adjacent to this trailhead.
kiosk string This field allows for data contributors to indicate a relationship between Named Trails that share constiuent segments. The anticipated use case is to indicate that a trail serves a leg in a larger regional or national trail.
osmTags string-encoded array This field is designed to incorporate a fungible number of key-value pairs that describe the trailhead in OpenStreetMap, such as arbitrary amenties.

areas.geojson

Name Type Req'd? Summary Values
geometry polygon or multipolygon yes The geometry of the area.
name string yes The official or publicly-advertised name of the land area.
id string yes A unique ID for this land area, which can be referenced.
stewardId string yes This unique ID associates each land area with the primary steward responsible for visitors within its boundaries.
url string HTTP link to information specifically about this land area.
osmTags string-encoded array This field is designed to incorporate a fungible number of key-value pairs that describe the trailhead in OpenStreetMap, such as arbitrary amenties.
